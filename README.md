## Реализация Машины Тьюринга на Node.JS

### Особенности
- **Гибкая настройка:** Все параметры машины задаются в JSON-файле, который нужно создать в каталоге `machines`.
- **Визуализация:** Анимированный вывод в консоль. При каждом шаге показывается, какое именно правило было применнено.
- **Простота:** Написан на чистом JavaScript (ES Modules) c единственной зависимостью - `chalk`.

---

### Требования
- Node.JS версии 14.x или выше.
- npm

---

### Установка
1. Клонируйте репозиторий на свой компьютер:
    ```bash
    git clone https://github.com/KirieshkaForever/turing-machine.git
    cd turing-machine
    ```

2. Установите все зависимости:
    ```bash
    npm install
    ```

---

### Запуск

Запускается из командной строки. В качестве аргумента нужно передать название JSON-файла.

```npm
node index.js <машина.json>
```

В комплекте идет пример doubler.json, который удваивает количество единиц на ленте.

```npm
node index.js doubler.json
```

### Формат конфигурационного файла

Для описания машины используется JSON-файл со следующей структурой:

```json
{
  "name": "Название вашей машины",
  "description": "Краткое описание того, что она делает",
  "blank": "_",
  "initialState": "start",
  "finalStates": ["halt"],
  "tape": "111",
  "rules": [
    { "state": "...", "read": "...", "write": "...", "move": "...", "nextState": "..." },
    { "state": "...", "read": "...", "write": "...", "move": "...", "nextState": "..." }
  ]
}
```

Описание полей:

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | `String` | Название машины. Не используется в логике, но полезно для документации. |
| `description` | `String` | Краткое описание. |
| `blank` | `String` | Символ, обозначающий пустую ячейку на ленте. |
| `initialState` | `String` | Имя начального состояния, в котором машина находится при запуске. |
| `finalStates` | `Array` | Массив строк с именами конечных (завершающих) состояний. При попадании в одно из них машина останавливается. |
| `tape` | `String` | Начальное содержимое ленты. Головка по умолчанию устанавливается на первый символ. |
| `rules` | `Array`| Массив объектов, описывающих правила перехода. |

**Структура правила:**

Каждый объект в массиве rules описывает одно правило перехода и имеет 5 полей:
- **`state`:** Состояние, в котором должно сработать правило.
- **`read`:** Символ, который головка должна прочитать на ленте, чтобы правило сработало.
- **`write`:** Символ, который будет записан в текущую ячейку ленты.
- **`move`:** Направление движения головки (L - влево, R - вправо, N - не двигаться).
- **`nextState`:** Новое состояние, в которое перейдет машина после выполнения правила.

### Принцип работы Машины Тьюринга

Машина Тьюринга — это абстрактная вычислительная машина, предложенная Аланом Тьюрингом в 1936 году. Она является теоретической моделью компьютера и способна имитировать любой алгоритм.

Текущая реализация эмулирует ключевые компоненты этой модели:

**Основные компоненты**
1. **Бесконечная лента (`Tape`):** Фундаментальный компонент абстрактного вычислительного устройства, представляющий собой линейную последовательность ячеек, которая простирается бесконечно в одну или обе стороны.
2. **Считывающая/записывающая головка (`Head`):** Это механизм, который может перемещаться по ленте влево (`L`) или вправо (`R`). Головка умеет выполнять три действия:
    - **Читать символ из ячейки, над которой она находится.**
    - **Записывать новый символ в эту же ячейку, затирая старый.**
    - **Перемещаться на одну ячейку влево или вправо.**
3. **Набор состояний (`States`):** Машина в любой момент времени находится в одном из заранее определенных состояний. Среди них есть:
    - **Начальное состояние (`initialState`):** Состояние, с которого начинается работа.
    - **Конечные состояния (`finalStates`):** Состояния, при попадании в которые машина завершает свою работу.
4. **Книга правил (`RuleBook`):** Ключевой элемент во всей машине. Он содержит инструкции (правила перехода), которые определяют, что делать в той или иной ситуации. Каждое правило имеет вид:
    > Если ты в состоянии `А` и головка читает символ `X`, то запиши символ `Y`, сдвинь головку вправо/влево и перейди в состояние `Б`.

**Цикл работы машины**

Машина работает пошагово, повторяя один и тот же цикл, пока не достигнет конечного состояния или не столкнется с ситуацией, для которой нет правила.

Этот процесс можно представить в виде следующей схемы:

```mermaid
graph TD
    A[Начало] --> B{Находимся в конечном состоянии?};
    B -- Да --> F[Успешное завершение];
    B -- Нет --> C[Считать символ под головкой];
    C --> D{Найти правило для (текущее состояние, считанный символ)};
    D -- Правило найдено --> E[Выполнить действия из правила];
    E -- 1. Записать новый символ --> G[2. Передвинуть головку (L/R)];
    G --> H[3. Перейти в новое состояние];
    H --> B;
    D -- Правило не найдено --> I[Аварийная остановка];
```

**Объяснение**

1. Машина проверяет, не находится ли она уже в одном из конечных состояний. Если да - работа успешно завершена.
2. Если нет, головка считывает символ на ленте.
3. Машина ищет в своей книге правил инструкцию, которая соответствует текущему состоянию и считанному символу.
4. **Если правило найдено**, машина выполняет три действия: записывает новый символ, двигает головку и переходит в новое состояние. После этого цикл начинается заново.
5. **Если правило не найдено**, машина не знает, что делать дальше, и аварийно останавливается. В текущей реализации это сопровождается сообщением "не найдено подходящее правило".
